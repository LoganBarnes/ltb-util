Template Class GenericGuard

  • Defined in File generic_guard.hpp

Class Documentation

template<typename Init, typename Destroy, typename ...Args>
class ltb::util::GenericGuard

Serves as a generic way to handle push/pop style calls using RAII.

This class creates a guard that calls the init function on construction and the destroy function on destruction: 

{
    auto guard = ltb::util::make_guard(&push, &pop); // calls push() here on creation
    ...
    // other code
    ...
} // calls pop() here when the guard goes out of scope

If both functions accept the same arguments or one function takes arguments and the other does not, the function pointers can be used directly and the arguments can be passed in after: 

auto guard1 = ltb::util::make_guard(&needs_a_string, &needs_the_same_string, "a string for both functions");
...
auto guard2 = ltb::util::make_guard(&needs_double, &needs_nothing, 3.14158265359);
...
auto guard3 = ltb::util::make_guard(&needs_nothing, &needs_two_args, 42, "this is cool");

If both functions need differing arguments then lambdas can be used: 

auto guard4 = ltb::util::make_guard([] { push(arg1, arg2); }, [] { pop(arg3); );
...
auto guard5 = ltb::util::make_guard([] { push(arg1, arg2); }, &pop, arg3);

Template Parameters

  • Init – is the function called on creation

  • Destroy – is the function called on destruction

  • Args – are the optional arguments used by the Init and Destroy functions
